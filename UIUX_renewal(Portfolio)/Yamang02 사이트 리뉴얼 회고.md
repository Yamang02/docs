## 고민하고 있었던 문제점

- 한 페이지 안에 지나치게 정보가 많다
	- 개발한 사람조차 처음에 보면 뭘 해야 하나 싶다 (저니맵?)

- 전형적인 AI 컬러파레트

- 더 이상 Gemini API를 통해 원활한 채팅 서비스 제공이 힘들다
	- 2025년 12월 정책변경으로 무료 사용량을 기존 1/10 수준으로 제한
	- 사이트의 메인 컨셉이었으나, API호출의 리스크 증가 

## 원림님 피드백 정리

  A. 사이트 목적과 전략을 구체화한 후에 디자인
  B.  목적성 있는 기능사용(AI 챗봇)
  C. 메인페이지 퍼스널 브랜딩
  D. 유사한 성격의 정보는 가까이 배치
  E. 의도된 정보에 집중할 수 있도록 요소 재배치/제거
  F. 자연스럽게 기능을 경험하는 흐름 구성 추천

## 리뉴얼 기획

0. 모든 패널형 UI 제거 (A, E)
	- 모바일, 태블릿, 브라우저에서 동작을 신경써야 할 요소가 많았음


1. 페이지로 정보 분리 (C, D)
	- '나'를 알릴 수 있는 프로필 페이지 추가
		- 기존 페이지의 교육/경력을 이동
	- 채팅 봇 패널을 페이지로 변경 (B, F)
		- 초기 의도는 모든 페이지에 고정되어 있는 상태로 '나'를 설명하는 비서역할이었으나, 기술/비용적으로 사이트의 중심 컨셉이 되기는 어려움. 
		- 추후 고도화된 기술 구현을 위해 기능 자체는 유지하지만, 별도로 독립적인 페이지로 구성하여 언제든 들어낼 수 있도록(UI적으로 사용자의 접근을 막도록) 구성

2. 디자인 시스템 구축 (C)
	- 컬러파레트, 재사용컴포넌트들을 체계적으로 관리


3. 랜딩페이지 신설(A, F)

- 관련 피드백
![프로젝트 목록 페이지 피드백](screenshots/Pasted%20image%2020260108100929.png)

- 필터 기능은 별도의 모달로 분리하고, 결과를 목록으로 제공해 바로 상세페이지로 연결될 수 있도록 구성
	- 에이전틱 코딩의 특성으로 인해, 고전적인 개발자의 포트폴리오에서 중시되던 '어떤 기술스택을 사용하였나'는 성격이 많이 약해짐.
	- 프로젝트를 필터링할 사람은 기술자가 유력. 기술스택으로 필터링이 가능하도록 모달 구성

![프로젝트 필터 모달](screenshots/Pasted%20image%2020260108101242.png)

- 어떤 도메인의 서비스를 만들었나가 중심이 되어야 할 것으로 고민 중
	- 기술 중심이 아니라 서비스 중심
- 가장 보여주고 싶은 프로젝트는 랜딩페이지에서 1차적으로 노출하고 프로젝트 페이지의 별도의 섹션과 배지로 강조
	- 다른 섹션들도 프로젝트의 성숙도에 따라 섹션을 분리해 사용자의 필터링과 정렬을 기대하는 것이 아니라 '내'가 의도한 순서에 따라 배치

- 히스토리 패널은 삭제하고, 프로젝트의 순서를 고려한 가로형 타임라인 섹션으로 변경![타임라인 섹션](screenshots/Pasted%20image%2020260108103459.png)
	- 정확한 일자와 기간 시각화보다, 프로젝트들의 순서로 성장과정을 보여주는 것이 더 합리적이라고 판단. 간단한 연도 라벨만 붙이는 것으로 단순화
